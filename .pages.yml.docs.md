# Pages CMS Configuration Documentation

## Overview

This document explains the `.pages.yml` configuration file for Pages CMS, which provides a complete content management system for the Next.js project. The configuration defines media handling, reusable components, content collections, and global settings.

## Media Configuration

The media section defines how to handle file uploads for images, documents, and other media types.

### Configured Media Folders

1. **Images** (`/media/images`)
   - Input: `public/media/images`
   - Output: `/media/images`
   - Categories: Images only (jpg, jpeg, png, gif, svg, bmp, tif, tiff)

2. **Documents** (`/media/documents`)
   - Input: `public/media/documents`
   - Output: `/media/documents`
   - Categories: Documents and PDFs (pdf, doc, docx, ppt, pptx, xlsx, txt, rtf)

3. **General Media** (`/media`)
   - Input: `public/media`
   - Output: `/media`
   - All file types

## Components

Components are reusable field groups that can be referenced in content entries. This project includes five pre-built components:

### 1. SEO & Meta Component

Provides SEO metadata for content entries.

**Fields:**
- `metaTitle` (string): Title for search engines and social sharing
- `metaDescription` (text): Description for search engines (150-160 chars)
- `keywords` (string): Comma-separated keywords
- `ogImage` (image): Image for social media sharing

### 2. Author Component

Defines author information for blog posts and other content.

**Fields:**
- `name` (string, required): Author's full name
- `email` (string): Author's email with validation
- `bio` (text): Author biography
- `avatar` (image): Author profile image

### 3. Hero Component

Reusable hero section for pages and content blocks.

**Fields:**
- `title` (text, required): Main title
- `subtitle` (text): Subtitle or tagline
- `backgroundImage` (image): Hero background image
- `ctaText` (string): Call-to-action button text
- `ctaLink` (string): Call-to-action button URL

### 4. Content Block Component

Structured content blocks with flexible layout.

**Fields:**
- `heading` (string, required): Block heading
- `content` (rich-text): Content body with rich text formatting
- `image` (image): Associated image
- `imagePosition` (select): Position of image (left, right, top, bottom)

### 5. Navigation Item Component

Reusable navigation menu items.

**Fields:**
- `label` (string, required): Display label
- `href` (string, required): URL or path
- `target` (select): Link target (_self, _blank, _parent, _top)

## Content Collections

### Blog Posts Collection

**Path:** `src/content/blog`
**Format:** YAML Frontmatter
**Filename Pattern:** `{year}-{month}-{day}-{primary}.md`

**Fields:**
- `title` (string, required): Post title
- `slug` (string, required): URL-friendly identifier
- `excerpt` (text, required): Short summary
- `author` (object, required): Author info (uses author component)
- `publishedAt` (date, required): Publication date
- `updatedAt` (date): Last updated date
- `featured` (boolean): Mark as featured post
- `tags` (string array): Post tags (1-10)
- `categories` (select array): Content categories (1-3)
  - Technology
  - Design
  - Business
  - Lifestyle
  - Other
- `coverImage` (image): Post cover image
- `seo` (object): SEO metadata (uses seo component)
- `body` (rich-text, required): Post content

**Collection View:**
- Displays: title, author.name, publishedAt, featured
- Sortable by: publishedAt, title
- Searchable: title, excerpt, tags, body
- Default sort: publishedAt (descending)

### Pages Collection

**Path:** `src/content/pages`
**Format:** YAML Frontmatter
**Filename Pattern:** `{slug}.md`

**Fields:**
- `title` (string, required): Page title
- `slug` (string, required): URL slug (lowercase, hyphens only)
- `description` (text): Page description
- `published` (boolean): Publication status
- `order` (number): Menu order
- `showInNav` (boolean): Show in navigation menu
- `sections` (array): Page sections with type and data
  - Section types: hero, content, features, testimonials, cta
  - Data stored as JSON code block
- `seo` (object): SEO metadata (uses seo component)
- `body` (rich-text): Main page content

**Collection View:**
- Displays: title, slug, published, order
- Sortable by: order, title
- Searchable: title, description, slug
- Default sort: order (ascending)

### Projects/Portfolio Collection

**Path:** `src/content/projects`
**Format:** YAML Frontmatter
**Filename Pattern:** `{primary}.md`

**Fields:**
- `title` (string, required): Project title
- `description` (text, required): Project description
- `thumbnail` (image): Project thumbnail
- `images` (image array): Project screenshots (1-20)
- `technologies` (string array): Tech stack used (required)
- `startDate` (date): Project start date
- `endDate` (date): Project completion date
- `status` (select): Project status
  - Completed
  - In Progress
  - On Hold
  - Archived
- `url` (string): Live project URL
- `gitHubUrl` (string): GitHub repository URL
- `featured` (boolean): Mark as featured project
- `body` (rich-text): Project details

**Collection View:**
- Displays: title, status, startDate, featured
- Sortable by: startDate, title
- Searchable: title, description, technologies
- Default sort: startDate (descending)

### Site Settings File

**Path:** `src/data/settings.json`
**Format:** JSON

Global site configuration including:
- `siteName`, `siteDescription`, `siteUrl`
- Author information
- Logo and favicon
- Navigation menu items with optional submenus
- Social media links
- Google Analytics configuration

### Navigation File

**Path:** `src/data/navigation.json`
**Format:** JSON

Main navigation structure with items, labels, URLs, and ordering.

## Global Settings

The global settings section controls Pages CMS UI and behavior:

```yaml
settings:
  hide: false                    # Show/hide settings in Pages CMS UI
  content:
    merge: false                 # Merge form data with existing file (don't overwrite)
```

**Current Configuration:**
- Settings page is visible
- Files are completely overwritten on save (merge disabled)

## Usage Tips

### Adding New Content

1. **Blog Post**: Navigate to Blog > Add New. Fill in metadata, write content in body.
2. **Page**: Navigate to Pages > Add New. Configure layout using sections array.
3. **Project**: Navigate to Projects > Add New. Upload images and fill project details.

### Creating Media Folders

Media folders are automatically created when you first upload files. Ensure folders match the configured paths in media section.

### Editing Components

To modify components, edit `.pages.yml` directly. Changes apply immediately on refresh.

## Field Type Reference

All available field types used in this configuration:

- **string**: Single-line text
- **text**: Multi-line text
- **rich-text**: Formatted text with bold, italic, links, etc.
- **image**: Image file upload
- **date**: Date picker
- **boolean**: Toggle switch
- **select**: Dropdown with predefined options
- **number**: Numeric input
- **object**: Complex nested structure
- **code**: Code editor (language-specific)

## Notes

- All file paths are relative to the repository root
- Collection filenames are generated from field values and date tokens
- Search functionality uses Lunr.js for full-text search
- Image and media paths use `/` prefix for absolute CDN URLs
- Components are inherited using the `component:` key in field definitions
